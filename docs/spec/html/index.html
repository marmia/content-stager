1. content-stager Docs

このリポジトリのゴールは「実装に繋がるドキュメント類」を整備し、 Codex（将来的に OpenCode）と Agent Skills を中心に MVP を組み立てること。

1.1. 入口（読む順）

  1. spec/concepts.org（設計思想・用語・状態モデルの原型）
  2. spec/decisions.org（確定事項のログ）
  3. spec/mvp.org（MVP でやること／やらないこと）
  4. spec/flow.org（URL 投入→S1→S2 の処理フロー）
  5. spec/content-fetch.org（URL→本文取得の契約）
  6. spec/data-model.org（SQLite のデータモデル/DDL）
  7. spec/skills.org（Skills 駆動実装の方針・ I/O 契約）
  8. spec/open-questions.org（未決事項）
  9. spec/ingesturl.org（ingesturl の詳細仕様）
  10. spec/cli.org（UI/CLI 契約）
  11. spec/testing.org（テスト指針）
  12. spec/shelves.org（Shelf 仕様）
  13. spec/recall.org（Recall 仕様）
  14. spec/signals-and-issues.org（強シグナル/Issue 定義）
  15. spec/commands.org（コマンド一覧）
  16. spec/topic-labeling.org（Topic 意図ラベル）
  17. spec/scoped-reveal.org（限定可視）
  18. spec/storage-ops.org（SQLite 運用）
